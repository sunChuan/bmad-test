## Dev Notes

### Technical Requirements
- **Frontend Stack**: TypeScript 5.x, Vue 3 (Composition API), Vite, Tailwind CSS, Ant Design Vue (内嵌在 Vben中).
- **Backend Stack**: Java 21, Spring Boot 3.2.x, MyBatis-Plus, Spring Security 6.x, MySQL 8, Redis 7.
- **RESTful API Standard**: MUST follow `/{api-version}/{module}/{resource}/{id}` (e.g., `/api/v1/assessment/schools/{id}`). Non-standard URIs are strictly forbidden.
- **Global Response Wrapper**:
  ```json
  {
    "code": 200,
    "message": "success",
    "data": { ... }
  }
  ```
- **Time Format Constraint**: Cross-language timestamps MUST be transmitted as fixed formatted strings `YYYY-MM-DD HH:mm:ss`, NO timestamp longs.

### Architecture Compliance
- Backend must follow strict layered DDD separation: Controller -> Service -> Repository. Controller has zero business logic; Repository only handles SQL.
- **Naming Conventions**:
  - MySQL Database Tables & Columns: Snake case WITH module prefix (`edu_` prefix expected, e.g., `edu_school_record`).
  - Java Entity/DTO: PascalCase class names, camelCase fields.
  - Vben Components: PascalCase filenames (e.g., `SchoolRadarChart.vue`).
- Avoid any data-layer queries inside loops; respect database performance NFRs.

### File Structure Requirements
You MUST adhere strictly to the architected physical bounds:
- `/frontend`: All Vben Vue3 Vite code.
- `/backend`: All Spring Boot Java 21 code.
- `/deploy`: Infra automation scripts (Docker).
DO NOT mix backend configurations into the frontend directory or vice versa.

### Design & UX Requirements
- **Desktop-First Strategy**: The main app wrapper should be built for 1280px+ resolution layout primarily.
- Main theme color: Deep Slate Blue (`#1E293B`) for overarching app layout containers.

### Testing Requirements
- Backend: JUnit 5 test context must be loadable without hard failure.
- Frontend: Vitest must run without crashing.
