# Story 2.1: 初始化市区级宏观热力图缓存网关

Status: ready-for-dev

## Story

As a **市级管理员** (或区县管理员),
I want **登录主页后能瞬间 (<3秒) 看到辖区内的增值评价雷达图和总体分数流向图**,
so that **能够对总体教育生态有一个立刻的宏观把握 (FR10, NFR5)**。

## Acceptance Criteria

1. **Given** 数据库中有大量的历年成绩且计算过程极具消耗
2. **When** 前端页面初始化发起 `GET /api/v1/dashboard/macro/heat-map` 请求
3. **Then** Java API 网关直接命中 Redis 中的前置聚类 JSON 缓存，并在极短时间（<500ms）内返回数据。绝不允许在此接口中进行任何扫表或连表查询。
4. **And** 返回的 JSON 数据结构中必须包含一个标志位组，用于在前端提示该数据的“最后刷新时间”以及“人工修正数据的占比率” (FR10.1)。

## Dev Notes

### Architecture Parameters & Constraints
- **性能约束 (NFR5 & NFR9)**: API P95 响应时间必须小于 500ms，必须只走 Redis 缓存获取结果，Java 作为单纯的只读路由网关。
- **权限安全**: 需融合 Epic 1 完成的 `TenantContextHolder` 获取多租户上下文，决定拉取全市数据还是单独区县的数据。
- **数据格式**: 根据架构规范，统一使用 `{ code: 200, message: "success", data: {...} }` 的 `Result<T>` 接口进行包装。

### Source Tree Components to Touch
- `backend/pom.xml`: 确保 Redis 依赖开启。
- `backend/src/main/java/com/bmad/edge/config/RedisConfig.java`: 可选（若需自定义序列化缓存配置）。
- `backend/src/main/java/com/bmad/edge/controller/DashboardController.java`: API 网关接入点。
- `backend/src/main/java/com/bmad/edge/service/DashboardService.java`: 缓存抓取逻辑。
- `backend/src/main/java/com/bmad/edge/dto/...`: 传输结构体。
